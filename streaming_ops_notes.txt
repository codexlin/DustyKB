DustyKB 流式问答运维说明

本文档用于测试知识库系统的流式输出、检索引用和中文问答能力。

一、流式输出

DustyKB 的流式问答接口使用 Server-Sent Events，也就是 SSE。后端会先返回 sources 事件，告诉前端本次回答引用了哪些文档片段；随后不断返回 token 事件，让前端把答案逐字或逐段显示出来；最后返回 done 事件，表示本次回答结束。

这种方式的优点是用户不需要一直等待完整答案生成完成。即使模型回答较长，页面也能更快给出反馈。

二、查询日志

每次成功问答后，系统会把问题、答案、引用来源、模型名称、耗时和创建时间写入 PostgreSQL。前端底部的“问答历史档案”面板会读取这些记录。

三、常见排查

如果页面没有流式显示，优先检查后端是否正在运行新版代码，并确认 /api/query/stream 接口可访问。

如果上传文档失败，优先检查 DashScope API Key、Qdrant 服务和 PostgreSQL 服务是否正常。

如果回答没有引用来源，可能是当前知识库没有文档，或者问题与文档内容相关性太低。

四、建议测试问题

1. DustyKB 的流式接口会返回哪些事件？
2. 查询日志会保存哪些字段？
3. 如果页面没有流式显示，应该先检查什么？
